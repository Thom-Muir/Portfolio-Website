CSUSB Capstone Project

Overview

As part of a rigorous capstone project, my team and I were challenged to apply everything we had learned throughout our bachelor's program. Our task: design and implement an enterprise-grade network supporting e-commerce services—from scratch, in just eight weeks. The final system underwent a week-long functional evaluation by the "Orange Team" while simultaneously being stress-tested by a live penetration test from the "Red Team."

Network Architecture

Our network was structured into three key segments: LAN, Screened Subnet, and WAN, connected via static routing and secured by a pfSense firewall. The following diagram illustrates the final architecture:

The firewall was configured to allow only HTTPS and SQL traffic into the screened subnet while blocking all unauthorized access to the LAN. This ensured secure communication and protected critical resources from external threats.

E-Commerce Services Deployment

The foundation of the our e-commerce services was our windows server. The tool XAMPP was installed for easy service and dependency management for Apache, MySQL, and PHP.

To better secure our database, the account with write access was restricted to localhost only to prevent external hijacking. Additionally both Apache and SQL connections where configured to use TLS to prevent on-path attacks.

The remaining RHEL server was repurposed to run a Python-based honeypot, designed to mislead attackers and slow down penetration testers.

User Authentication & Access Control

Active Directory (AD) was configured to allow for network authentication of LAN clients. This allowed for the centralized management of group policy and user accounts from the AD server.

The Orange Team was given a read-only restricted account to access information from the e-commerce SQL server.

The AD server’s host based firewall was configured to only whitelisted devices to establish connections, disallowing unauthorized IP addresses from attempting login on the LAN.

Email and Customer Support

An internal email service was created to allow Orange team users to file support tickets during the technical evaluation. This service was integrated with the existing AD accounts for consistent Identity & Access Management.

Email protocols used where secured using TLS to prevent attacks from intercepting communications.

Created an HTML-based guide with step-by-step instructions for accessing network resources from client machines.

Results & Takeaways

  • Achieved an A in the course with a fully functional and secure network.
  • Successfully defended against penetration testing with no vulnerabilities exploited and all incidents logged.
  • Orange Team reported high satisfaction with prompt and professional support.
  • Gained hands-on experience in enterprise networking, security, and server administration.
